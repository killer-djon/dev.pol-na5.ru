Обзор WebAsyst Фото

WebAsyst Фото — это приложение WebAsyst, предназначенное для хранения и публикаций фотографий. WebAsyst Фото позволяет организовать фотографии по альбомам, коллективно работать с фотографиями и красиво публиковать фотографии в интернете. 

На главном экране WebAsyst Фото отображается список альбомов с фотографиями:

Все изображения в WebAsyst Фото организованы по альбомам.

Фото не может находиться вне альбома — сначала необходимо создать альбом, затем можно загружать в него фотографии.

У альбомов нет иерархии: есть только плоский список альбомов. Нельзя один альбом сделать «подальбомом» другого.

Порядок сортировки альбомов задается в режиме «Управление альбомами», перейти в который можно по одноименной ссылке в правом верхнем углу главного экрана WebAsyst Фото. Перетаскивая альбомы с помощью мышки в режиме управления, можно задавать порядок отображения альбомов.

Физически на сервере для каждого альбома создается выделенная папка, в которую помещаются файлы фотографий. 

WebAsyst Фото состоит из двух основных частей: 

  • режим управления (бекенд): часть WebAsyst Фото, требующая авторизации пользователя (входа в WebAsyst); это то, что вы видите, когда заходите в WebAsyst в приложение «Фото».

    В этом режиме производится все управление фотографиями: можно загружать фотографии, создавать альбомы, публиковать альбомы и т.д. Доступ к режиму управления фотографиями настраивается для каждого пользователя WebAsyst на уровне альбомов и основных экранов. Подробнее о правах доступа и коллективной работе с фотографиями рассказано ниже в главе «Коллективная работа».

  • общедоступная часть (фронтенд): часть WebAsyst Фото, не требующая авторизации;

    Пример: http://demo-ru./photos/
    В общедоступной части отображаются только опубликованные альбомы. Подробнее о публикации альбомов рассказано ниже в главе «Публикация фотографий».

Загрузка фотографий

Сразу много и по одной


В WebAsyst Фото поддерживается два способа загрузки фотографий:

  • сразу много: можно выбрать сразу до 500 файлов и нажать кнопку «Загрузить» — все файлы будут загружены автоматически по очереди; для работы этой функции необходим установленный Adobe Flash Player;
  • по одной: каждый файл выбирается отдельно в диалоге загрузки; загрузка работает и без установленного Adobe Flash Player;

Эскизы и оригиналы фотографий

При загрузке фотографии в WebAsyst Фото ее оригинал сохраняется в защищенной от общего доступа папке на сервере. Доступ к оригиналу фотографии есть только по ссылке, которую можно получить в режиме администрирования, имея доступ к альбому. Подобрать ссылку практически невозможно (для этого потребуется подобрать хеш-код длиной 32 символа).

При загрузке (после сохранения оригинала в защищенной папке) для каждой фотографии автоматически создаются пять эскизов:

  • 96х96 (кроп),
  • 144х144 (кроп),
  • 256-,
  • 512-,
  • 750-,
  • 970- пикселей по большему измерению (размер 970 пикселей выбран для того, чтобы ваши опубликованные альбомы правильно отображались при разрешении экрана 1024х768 у посетителей вашей фотогалереи).

Эти эскизы используются при отображении фотографий: при просмотре, публикации, отправке ссылки на фото, экспорте фотографий. Эскизы занимают значительно меньше пространства на диске, чем оригинал, и поэтому передаются через интернет быстрее.

Размеры эскизов не настраиваются — они явно заданы в WebAsyst Фото и не могут быть изменены без изменения исходного кода (PHP-скриптов).

Ограничение на размер загружаемой фотографии в веб-сервисе WebAsyst Фото — 20 мегапикселей. Если вы пользуетесь скриптами, такого ограничения нет — максимальный размер фотографии ограничен только возможностями вашего сервера (в первую очередь, объемом памяти, отведенной PHP-скриптам).

Настройка создания эскизов 

Создание эскизов производится с помощью одного из расширений PHP для работы с графикой: Imagick или GD. Какое из расширений используется, настраивается в разделе «Настройки» WebAsyst Фото.

Если Imagick поддерживается вашим сервером, то рекомендуем использовать именно Imagick, потому как это расширение дает более качественные результаты: использует меньше памяти, создает эскизы быстрее, эскизы получаются лучшего качества.
Посмотрите подробный ответ на вопрос «Что лучше: GD или Imagick?» в ответах на вопросы о WebAsyst Фото.

Опция в настройках «Применять фильтр резкости (sharpen)» позволяет повысить качество эскизов фотографий. Включите эту опцию, чтобы фильтр резкости автоматически применялся при создании эскизов. Мы рекомендуем включить эту опцию.

 

Импорт из Flickr

WebAsyst Фото позволяет импортировать фотографии из вашего аккаунта в сервисе Flickr.

Для импорта фотографий необходимо создать альбом или войти в существующий альбом и щелкнуть по ссылке «Импорт из Flickr». В появившемся новом окне щелкните по ссылке «Настройки Flickr API», и вам будут показаны подробные инструкции о том, что необходимо сделать для импорта фотографий (как правильно настроить параметры Flickr API).

После настройки Flickr API щелкните по ссылке «Импортировать» и выберите набор фотографий (photo set). WebAsyst Фото позволяет импортировать фотографии только из наборов фотографий (если в вашем аккаунте нет наборов, их необходимо предварительно создать).

ВАЖНО: WebAsyst Фото импортирует фотографии из Flickr максимального размера, который предоставляет (отдает) Flickr. Как правило, это эскизы размера 1024-пикселей, однако это зависит от вашего аккаунта и фотографий.

Режим управления фотографиями

Работа с фотографиями внутри альбома

Внутри альбома доступны два основных режима работы:

  • режим просмотра фотографий («Просмотр»): щелкните по фотографии, чтобы перейти к ее просмотру;
  • режим управления фотографиями («Управление и ссылки»): задайте порядок сортировки фотографий перетаскиванием с помощью мышки (изменения сохраняются автоматически после перетаскивания фото), выберите отдельные фотографии, чтобы получить ссылки на эти фотографии, код для вставки выбранных фотографий в форум (BBcode) или на сайт (HTML), создать коллекцию (ссылку или виджет);

Основные свойства альбома:

  • название альбома (щелкните по названию альбома для редактирования):


  • подзаголовок: любой текст, который будет выводиться в общедоступной части под названием альбома; например, можно ввести даты когда был сделан альбом, кем или где был сделан альбом и т.п.; можно оставить это поле пустым; для редактирования подзаголовка щелкните по ссылке «Редактировать описание альбома»;
     
  • описание альбома выводится в общедоступной части (фотогалерее) под названием и подзаголовком альбома; в описании можно использовать HTML-теги;
  • обложка альбома (ключевая фотография альбома): фотография, которая будет поставлена на обложку альбома в режиме управления фотографиями (в бекенде) и в фотогалерее; для того, чтобы установить фотографию на обложку, в режиме просмотра фотографий щелкните по фото, а затем по ссылке «Поставить на обложу альбома» — WebAsyst Фото автоматически создаст два изображения, которые будут использоваться как обложки альбомов для бекенда и общедоступной фотогалереи.

Просмотр фотографии

 

В режиме просмотра одной фотографии доступны основные инструменты работы с фото:

  1. Выбор размера эскиза фотографии из всех доступных размеров эскизов: 96х96, 144х144, 256, 512, 750, 970. При изменении текущего размера эскиза автоматически обновляются ссылки на фото и код экспорта фотографии (см. следующий пункт).
    Если выбранный эскиз не вмещается в отведенное фотографии пространство в браузере (например, вы выбрали эскиз размера 970, а разрешение экрана установлено 1024х768 пикселей), то эскиз будет смасштабирован браузером только на отведенную область (т.е. вы увидите несколько уменьшенную версию эскиза).
  2. Экспорт (встраивание) фотографий в блоги или сайты (HTML) и форумы (BBcode), получение ссылок на фотографию для отправки по почте или IM-мессенджере. Скопируйте код и используйте его по назначению.
    Ссылки на одну или несколько фотографий будут работать вне зависимости от статуса альбома (о статусе альбомов читайте ниже в главе «Публикация альбомов») — альбом может быть неопубликованным, но при этом определенные фотографии вы можете отправлять друзьям или встраивать их на сайты и форумы.
  3. Оригинал: ссылка на загрузку оригинала изображения доступна только для пользователей, у которые есть доступ к альбому с фотографиями.
  4. Инструменты для работы с фото: поворот влево и вправо (по- и против- часовой стрелки), инструмент кадрирования (вырезания части фотографии, «кроп»; этот инструмент работает только когда текущий размер эскиза более 144 пикселей).
    ВАЖНО: При использовании этих инструментов изменяется оригинальное изображение. Это значит, что «откатить» выполненную операцию в случае использования инструмента кадрирования невозможно. В случае неудачно проведенной операции кадрирования придется заново загружать фотографию с вашего компьютера.

    Щелчок по инструменту «Поставить фото на обложку альбома» автоматически обновит изображения обложки альбома.

    Удаление фотографии производится с помощью щелчка по ссылке «Удалить фотографию». Действие необратимо.

  5. EXIF. Выводится имеющаяся информация о снимке. Например: модель фотоаппарата, выдержка, диафрагма, ISO и т.д.

Публикация фотографий

Фотографии публикуются в общедоступной части WebAsyst Фото (фотогалерее), которая доступна по адресу: http://ВАШ_АККАУНТ./photos/ (для пользователей веб-сервиса) или http://АДРЕС_УСТАНОВКИ_WEBASYST/photos/ (для пользователей скриптов).

Общедоступная часть не требует авторизации — любой посетитель фотогалереи может просмотреть все опубликованные альбомы.

Замечание для пользователей скриптов:

В WebAsyst Installer (административного приложения WebAsyst) в разделе «Настройки — Общедоступная часть» есть настройка, позволяющая настроить адрес работы вашей фотогалереи, а именно, указать, где будет открыться фотогалерея: [адрес_webasyst] или [адрес_webasyst]/photos/

Три статуса публикации альбома

Альбом фотографий в WebAsyst Фото может иметь один из трех статусов:

  1. Неопубликован: доступен только внутри администрирования WebAsyst Фото пользователям, имеющим соответствующие права доступа к альбому (по чтению, записи или полные права); о коллективной работе с альбомами читайте ниже в главе «Коллективная работа»;
  2. Опубликован: доступен из фотогалереи для всех посетителей, отображается в списке альбомов фотогалереи;
  3. Опубликован приватно: доступен из фотогалереи только по прямой ссылке — только для тех пользователей, кто знает прямую ссылку на этот альбом (в списке опубликованных этот альбом не появляется, ссылку на альбом вы можете сообщить друзьям и коллегам в частном порядке).

Приватная публикация очень удобна для частной презентации некоторых альбомов, при которой альбомы не публикуются в общем списке альбомов общедоступной части, но их можно посмотреть, зная ссылку на них. Авторизации с логином и паролем в этом случае не требуется.

Три вида представления фотографий альбома

Для каждого опубликованного альбома можно выбрать вид представления фотографий из трех вариантов:

  • вид №1: показывать фотографии и описания подряд на странице (хороший вариант, если фотографии сопровождаются исчерпывающими описаниями),
  • вид №2: текущая фотография и эскизы других фото справа (хороший вариант для альбомов с большим количеством фотографий, когда у фото либо нет описаний, либо они есть, но короткие),
  • вид №3: уменьшенные эскизы фотографий, увеличение по клику (хороший вариант для альбомов с большим количеством фотографий без описаний).

В видах №1 и 2 описания фотографий показываются так, как они заданы — с учетом тегов HTML. В виде №3 описание показывается только при увеличении фотографии, и HTML-теги вырезаются.

В видах №1 и 2 также доступна опция «Можно посмотреть и скачать оригинал фотографий альбома из галереи». Если эта опция включена, то для каждой фотографии при наведении на нее мышкой в общедоступной фотогалерее посетитель увидит ссылку «скачать оригинал фотографии» и, щелкнув по ней, сможет скачать оригинал любой фотографии альбома.

Коллекции фотографий

Коллекции фотографий — это еще один способ публикации фотографий. Коллекция представляет собой возможность объединить в группу несколько выбранных фотографий или фотографии всего альбома.

Для создания коллекции из всех фотографий альбома войдите в альбом и нажмите «Создать коллекцию». Чтобы создать коллекцию из нескольких выбранных фотографий, перейдите в режим «Управление и ссылки» внутри альбома и также нажмите «Создать коллекцию».

Коллекции бывают нескольких типов:

  • ссылка: прямая ссылка на фотографии, которую можно отправить по электронной почте или с помощью IM-мессенджера; когда того, как такой виджет создан, можно настроить вид представления фотографий (размеры эскизов, показывать или нет описания);
  • виджет «галерея»: HTML-код для встраивания на сайты и блоги, представляющий собой iframe, внутри которого в виде таблицы отображаются все фотографии коллекции с описаниями (с возможностью увеличения фотографий);
  • виджет «слайдшоу»: HTML-код для встраивания на сайты и блоги, представляющий собой iframe, внутри которого «крутится» слайдшоу фотографий коллекции.

Дизайн

Раздел «Дизайн» предоставляет возможность редактирования CSS-стилей общедоступной части (фотогалереи) и коллекций типа «Ссылка» — файл стилей, который редактируется в разделе «Дизайн» подключается и на страницах фотогалереи, и в коллекциях. Для коллекций-виджетов эти стили не подключаются.

Коллективная работа с фотографиями

Права доступа в WebAsyst Фото

В WebAsyst Фото права доступа пользователей регулируются на уровне альбомов. Предусмотрены три основные уровня прав доступа к альбому:

  • Чтение (Ч) – пользователь (группа пользователей) может просматривать изображения в альбома, просматривать слайдшоу, получить ссылки на фотографии,
  • Запись (З) – пользователь (группа) может как смотреть фотографии альбома (есть права по чтению), так и загружать новые изображения в альбом, изменять описания фотографий,
  • Полные (П) – пользователь (группа) имеет права по записи в альбоме, а также может управлять статусом публикации альбома в фотогалерее.

Права доступа регулируются в приложении WebAsyst Команда (необходимо иметь доступ к этому приложению).

Предусмотрены также несколько дополнительный прав доступа, регулирующих доступ к экранам и функциям WebAsyst Фото:

  • может создавать альбомы,
  • может управлять альбомами: определяет доступ к экрану управления (сортировки) альбомов и удаления альбомов,
  • может создавать и управлять коллекциями: определяет доступ к экрану «Коллекции» и функции создания коллекций фотографий,
  • может изменять дизайн: определяет доступ к экранам «Дизайн» и «Настройки».

Доступ к новым альбомам

Когда вы создаете новый альбом, он автоматически становится доступным только его автору (пользователю WebAsyst, который создал альбом).

Если вы хотите, чтобы при создании альбомов другие пользователи также получали доступ к новым альбомам автоматически (сразу после создания альбома), то это можно сделать, указав на экране «Настройки» сервиса WebAsyst Фото группы пользователей и значения прав доступа, которые нужно установить. После этого указанные права доступа будут автоматически устанавливаться для выбранных групп к каждому новому альбому.
